# MEMSHADOW Quick Start Guide

Get MEMSHADOW running in under 5 minutes.

---

## Option 1: Interactive Install (Recommended)

```bash
git clone https://github.com/SWORDIntel/MEMSHADOW.git
cd MEMSHADOW
./install.sh
```

The installer will guide you through:
- Admin account setup
- Database configuration
- Embedding backend selection (local or OpenAI)
- Security settings
- Feature toggles

---

## Option 2: Quick Install (Defaults)

```bash
git clone https://github.com/SWORDIntel/MEMSHADOW.git
cd MEMSHADOW
./install.sh --quick --run
```

Uses secure auto-generated passwords and recommended settings.

---

## Option 3: One-Command Docker

```bash
git clone https://github.com/SWORDIntel/MEMSHADOW.git
cd MEMSHADOW
cp config/.env.example .env
docker compose up -d
```

Default credentials: `admin` / `admin` (change immediately!)

---

## Option 4: Headless/CI Mode

```bash
export MEMSHADOW_ADMIN_PASS="YourSecurePassword123!"
export OPENAI_API_KEY="sk-..."  # Optional
./install.sh --headless --run
```

## Common Commands

```bash
# Stop
docker compose down

# Start
docker compose up -d

# Restart
docker compose restart

# View logs
docker compose logs -f

# Shell access
docker compose exec memshadow bash

# Run migration
docker compose run --rm memshadow python scripts/migrate_embeddings_to_2048d.py
```

## Embedding Options

### Local (Free, Recommended)
```bash
EMBEDDING_BACKEND=sentence-transformers
EMBEDDING_MODEL=BAAI/bge-large-en-v1.5
EMBEDDING_DIMENSION=2048
```

### OpenAI (API-based)
```bash
EMBEDDING_BACKEND=openai
OPENAI_API_KEY=sk-...
EMBEDDING_DIMENSION=2048
```

## Troubleshooting

### Services won't start
```bash
docker compose logs
docker compose down -v  # Reset volumes
docker compose up -d
```

### Permission denied
```bash
chmod +x install.sh
sudo usermod -aG docker $USER
# Log out and back in
```

### Port already in use
```bash
# Check what's using port 8000
lsof -i :8000
# Or change port in docker-compose.yml
```
